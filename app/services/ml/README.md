# ML-модуль: сообщение → REC/RAG-память → LLM → решение

Этот пакет отвечает за первый рабочий контур ML-слоя.

## Зачем он нужен

API не должно напрямую отправлять пользовательский текст в LLM. Между API и моделью нужен слой orchestration:

1. получить сообщение клиента;
2. достать релевантную память компании;
3. собрать системные инструкции для роли менеджера по продажам;
4. отправить prompt в LLM-провайдер;
5. посчитать уверенность;
6. решить: можно автоответить или надо передать менеджеру.

## Текущий режим

Пока нет API-ключа и Qdrant, модуль работает на:

- `KeywordMemoryRetriever` — локальная keyword-память для smoke-тестов;
- `MockLLM` — заглушка LLM-провайдера.

Контракт останется тем же, когда появится реальный провайдер:

- retriever заменится на Qdrant/PostgreSQL;
- `MockLLM` заменится на YandexGPT/GigaChat/OpenAI-compatible provider;
- API endpoint `/api/v1/ml/answer` менять не потребуется.

## Основной endpoint

`POST /api/v1/ml/answer`

На вход:

- `tenant_id`;
- `message`;
- история диалога;
- настройки AI;
- опциональный override памяти для тестов.

На выход:

- `answer`;
- `confidence`;
- `decision`: `auto_reply` или `escalate`;
- `sources`;
- `provider`.
