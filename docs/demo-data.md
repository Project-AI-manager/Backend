# Демонстрационные данные

Команда сидирования добавляет локальный набор данных для разработки и ручной проверки MVP без
ML-модуля.

## Запуск

```bash
python -m app.db.seed
```

Перед запуском нужно поднять инфраструктуру и применить миграции:

```bash
docker compose up -d
alembic upgrade head
python -m app.db.seed
```

Проверить состав данных без подключения к базе можно так:

```bash
python -m app.db.seed --dry-run
```

## Демо-доступы

- Владелец: `owner@demo.ai-manager.local`
- Менеджер: `manager@demo.ai-manager.local`
- Пароль для обоих пользователей: `demo-password`

## Что создаётся

- Компания `ООО Север`.
- AI-настройки компании с mock-провайдером.
- Демо-тариф и счётчики использования.
- Один активный канал `telegram`.
- Три клиента Telegram.
- Три диалога с сообщениями.
- Два документа базы знаний и несколько чанков.
- Один кандидат для пополнения базы знаний.
- Одна ручная эскалация.

Max и Avito в этой задаче не сидируются.
